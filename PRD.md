# PRTSBoost — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称

PRTSBoost（PRTSBoost）

### 1.2 产品定位

面向游戏代肝客户端的轻量级选购工具，以"点餐式"交互帮助用户快速浏览、筛选、勾选代肝项目，实时计算总价并导出汇总图片。

### 1.3 目标用户

- 对代肝品类不熟悉、不清楚自己需求的新客户
- 需要对比套餐与单项价格、灵活组合的老客户

### 1.4 核心价值

- **可视化浏览**：层级树状结构展示所有代肝品类，一目了然
- **灵活组合**：支持套餐一键勾选 + 单项自由搭配
- **实时计价**：勾选即算，无需手动加减
- **一键出图**：汇总已选内容生成图片，用户自行截图保存，无需对接支付

## 2. 功能需求

### 2.1 页面结构

小程序共 3 个页面，通过底部 TabBar 切换：

| Tab 位置 | 页面名称 | 功能说明 |
| --- | --- | --- |
| 左 | 套餐 | 展示主推优惠套餐，适合"一键全包"型用户 |
| 中 | 单项 | 展示所有可单独购买的项目，按品类层级罗列 |
| 右 | 总览 | 按大类汇总所有项目及全包优惠价，提供全局视角 |

### 2.2 套餐页

- 数据源：`PRTSBoost/data/packages.js`
- 每个套餐以卡片形式展示，包含：
  - 套餐名称
  - 内容说明
  - 要求/备注（可为空）
  - 价格
  - 变体价格（如"不含全收集"的备选价格）
- 每张卡片右上角有圆形添加按钮，点击后直接加入购物车
- **套餐互斥规则**：
  - **套餐一至四（四号谷地开荒、武陵开荒、四号谷地毕业、武陵毕业）支持多选**，互不排斥
  - **套餐五（省心一条龙）与套餐一至四全部互斥**：选择套餐五时自动清除套餐一至四；选择套餐一至四中任一个时自动清除套餐五
  - 套餐五涵盖前四个套餐的全部内容，因此不可并存
- **变体切换（套餐三/四）**：
  - 套餐三（四号谷地毕业套餐）和套餐四（武陵毕业套餐）各有两个变体：默认（全收集）和不含全收集
  - 卡片底部显示变体切换标签（如"默认" | "不含全收集 — 620r"）
  - 选中变体后，卡片显示对应价格（原价以删除线展示），购物车同步更新
  - 同一套餐的两个变体视为同一套餐，变体切换不影响互斥逻辑

### 2.3 单项页

- 数据源：`PRTSBoost/data/items.js`
- 内容按 **三层树状结构** 展示（无冗余中间层，第一章/第二章保留进程组用于有意义的细分）：

```
大类（如"主线任务"）  ← 含全包优惠价 428r
├── 子类（如"第一章"）
│   ├── 叶子项（如"基地解围 — 5r"）
│   ├── 叶子项（如"谷地重启 — 5r"）
│   └── 叶子项 ...
├── 叶子项（如"序章 — 15r"）  ← 无子分类时直接挂叶子
└── 子类 ...
```

- 若子类下无有意义的进程组划分（如支线任务、探索类、基建类），叶子直接挂在子类下，不设冗余中间节点
- 共 11 大品类（总计 149 个叶子项）：主线任务 40、支线任务（重要任务）22、次要任务 23、角色故事 5、功能任务 0、常驻活动 6、基础探索类 12、至尊全包探索类 12、1.2下半新增 7、基建拉电线 11、地区拉滑索 11
- **探索类互斥规则**：
  - "基础探索类"和"至尊全包探索类"是同一区域的**两档**选择（基础档 vs 至尊档）
  - 同一区域（如四号谷地）只能选择一档：要么基础探索 178r，要么至尊探索 258r，不可同时勾选
  - 此互斥关系在数据层不强制，但用户应知晓并自行注意
- "功能任务"品类无实际可选项，作为占位展示（"遇到的顺手清"）
- "1.2下半新增"品类为版本参考，其中部分项目与主数据重复（如"再引春来系列"亦出现在支线任务中），用户勾选时应注意避免重复计价
- 每个节点（含非叶子节点）均包含四项属性：

| 属性 | 说明 |
| --- | --- |
| 名称 | 节点标题 |
| 内容 | 该节点覆盖的内容描述 |
| 要求/备注 | 前置条件、注意事项，可留白 |
| 价格 | 叶子项为单价，非叶子项为全包优惠价；按关计价叶子项为 `null`，使用 `unitPrice` |

- 每个节点前有 **圆形勾选按钮**：
  - 勾选父节点 → 自动勾选其下所有叶子节点（全选）
  - 取消父节点 → 自动取消其下所有叶子节点（全不选）
  - 部分子节点被勾选时，父节点按钮显示为半选态（`—`）
- 已勾选项高亮显示，价格实时累加至购物车
- **按关计价叶子项**（如密境行者）：
  - 显示为 `3r/关` 而非固定价格
  - 加入购物车默认数量为 1，可在购物车弹窗中 +/− 调节
  - 其父节点的全包优惠价不参与计算（因变量计价无法适用固定全包价）

### 2.4 总览页

- 数据源：`PRTSBoost/data/overview.js`
- 以分类列表形式展示 **品类全包优惠**：
  - 任务类：主线任务全包 428r、支线任务（重要）全包 208r、次要任务全包 123r、角色故事全包 98r（统一归属"任务"大类）
  - 常驻活动：蚀像寻遗全包 100r、密境行者全包 55r
  - 探索类：基础探索（四号谷地 178r / 武陵 228r）、至尊探索（四号谷地 258r / 武陵 318r）
  - 基建类：基建拉电线（四号谷地 58r / 武陵 58r）、地区拉滑索（四号谷地 78r / 武陵 88r）
- 每行展示类目名称 + 项目名称 + 价格 + 圆形添加按钮，点击直接加入购物车
- 品类全包项目加入购物车时，类型标记为 `category_package`，独立于树的叶子项
- 购物车总价计算时，`category_package` 项目直接累加入总价，不与树的叶子项冲突
- 套餐部分移至专属"套餐"Tab 页，避免与总览页重复

### 2.5 购物车

- **入口**：每个页面左下角的悬浮购物车按钮（圆形，带角标显示已选数量）
- **弹出层**：点击按钮后从底部滑出半屏弹窗
- **弹窗内容**：
  - 已选项目列表，按分组显示：
    - "套餐"组（来自套餐页的选中套餐）
    - "品类全包"组（来自总览页的品类全包项目）
    - 按大类分组的叶子项（来自单项页）
  - 每个叶子项显示名称 + 价格
  - **按关计价项目**：显示 `−` `数量` `+` 按钮，支持调整关卡数量，价格实时更新
  - 支持在弹窗内删除单项（点击 `✕`）
  - 底部固定栏显示 **总价**
- **全包优惠价计算逻辑**：
  - 遍历树的所有有价格的非叶子节点（category / subcategory），按深度降序排列
  - 若某节点下所有叶子均被选中，使用该节点的全包价，其下叶子不再单独计费
  - 含按关计价叶子的节点不参与全包价计算
  - 套餐和品类全包项目直接累加
- **弹窗底部按钮**："生成汇总图片"（购物车为空时禁用）

### 2.6 图片生成

- 点击"生成汇总图片"后：
  1. 前端根据当前购物车数据在离屏 Canvas 上渲染一张竖版汇总图
  2. 图片内容从上至下：
     - 标题："终末地代肝 — 选购清单"
     - 日期
     - 已选项目明细列表（按分组展示；已全选且有全包价的父节点折叠为单条汇总项，省略子节点详情）
     - 分隔线
     - 总计（标签左对齐，价格数值居中，分行避免重叠）
     - 底部提示文字："请截图保存此图片"
  3. 图片生成后全屏预览，支持上下滚动查看长图
  4. 用户可长按图片保存或截图保存
  5. 固定右上角 ✕ 按钮关闭预览，点击遮罩空白区域也可关闭
  6. 切换 Tab 页面时自动关闭图片预览（`onHide` 生命周期清理）

## 3. 数据格式规范

### 3.1 数据存储

代肝项目数据以 **JavaScript 模块** 形式存储于 `PRTSBoost/data/` 目录下，在构建时直接 `require` 引入，无需运行时 Markdown 解析。

原始 Markdown 参考文件存放于项目根目录（`终末地代肝_*.md`），作为数据权威来源。修改数据时应在 MD 源文件和 JS 模块中同步更新。

### 3.2 节点模型

```typescript
interface ItemNode {
  id: string;           // 唯一标识
  name: string;         // 名称
  content: string;      // 内容描述
  requirement: string;  // 要求/备注
  price: number | null; // 价格（元），非叶子节点为全包价或 null
  type: string;         // 节点类型: 'category' | 'subcategory' | 'process_group' | 'leaf' | 'package'
  children: ItemNode[]; // 子节点
  isLeaf: boolean;      // 是否叶子节点（最小可购买单位）

  // 按关计价扩展（可选，仅叶子节点）
  isPerUnit?: boolean;  // 是否为按关计价模式
  unitPrice?: number;   // 每关单价（如 3r/关），与 price 互斥
}
```

### 3.3 按关计价模型

部分项目（如密境行者）采用按关计价：用户选择关卡数量而非固定价格。

- `isPerUnit: true` 的叶子节点，`price` 为 `null`，使用 `unitPrice` 作为单价
- 购物车中此类项目默认数量为 1，用户可加减数量
- 包含按关计价叶子的父节点，其全包优惠价不生效（因变量计价无法适用固定全包价）
- 显示格式：`3r/关`，购物车中显示 `×数量`

### 3.4 文件清单

| 文件名 | 对应页面 | 说明 |
| --- | --- | --- |
| `PRTSBoost/data/packages.js` | 套餐页 | 5 个主推套餐 |
| `PRTSBoost/data/items.js` | 单项页 | 全部品类树形数据（11 大类） |
| `PRTSBoost/data/overview.js` | 总览页 | 18 条品类全包 + 5 条套餐汇总 |

## 4. 交互设计

### 4.1 设计参考

整体风格参考常用点餐小程序/App（如美团、饿了么点餐界面）：
- 清爽白底 + 品牌强调色
- 卡片式项目布局，圆角阴影
- 底部 TabBar + 悬浮购物车按钮
- 流畅的底部弹窗动画

### 4.2 关键交互规则

1. **父节点全选/全不选**：点击父节点的勾选图标，递归选中/取消所有后代叶子节点
2. **半选态**：部分叶子被选中时，父节点勾选图标显示半选（`—`）
3. **全包优惠价计算**：全选某父节点下所有叶子时，自动使用全包优惠价而非逐项累加；含按关计价叶子的节点不适用全包价
4. **套餐互斥**：套餐一至四可多选（互不排斥）；套餐五与套餐一至四全部互斥（选择套餐五→清除套餐一至四，反之亦然）
5. **探索类互斥约定**：同一区域的"基础探索类"和"至尊全包探索类"为二选一，由用户自行注意
6. **跨页面状态同步**：购物车数据为全局状态（`app.globalData`），三个页面共享同一份已选列表
7. **弹窗遮罩**：购物车弹窗打开时，背景页面不可操作
8. **按关计价数量调节**：勾选密境行者等按关计价项目后，在购物车弹窗中通过 +/− 按钮调整数量，默认 1，最小 1

## 5. 技术方案

### 5.1 技术栈

| 层级 | 技术选型 |
| --- | --- |
| 框架 | 微信小程序原生框架（WXML + WXSS + JS） |
| 状态管理 | `getApp().globalData`（购物车数据、选中套餐 ID） |
| 数据源 | JavaScript 模块（`PRTSBoost/data/*.js`），构建时直接引入 |
| 图片生成 | Canvas 2D API（离屏 Canvas + `wx.canvasToTempFilePath`） |
| 构建工具 | 微信开发者工具 |

### 5.2 项目结构

```
/PRTSBoost
├── app.js                  # 全局状态（cartItems、selectedPackageId）
├── app.json                # 路由、TabBar 配置
├── app.wxss                # 全局样式（CSS 变量、卡片、按钮等）
├── pages/
│   ├── packages/           # 套餐页（主推优惠套餐）
│   ├── items/              # 单项页（四层树形结构）
│   └── overview/           # 总览页（品类全包 + 套餐汇总）
├── components/
│   ├── tree-node/          # 树节点行组件（递归展开/勾选/半选态）
│   ├── cart-popup/         # 购物车底部弹窗组件（分组/删除/数量调节/总价）
│   └── image-preview/      # 图片预览组件（Canvas 渲染 + 长按保存）
├── utils/
│   ├── cart.js             # 购物车逻辑（增删改查、全包价计算、分组、数量调节）
│   └── canvas-renderer.js  # Canvas 图片生成（标题/日期/明细/总价/提示）
└── data/                   # JavaScript 数据模块
    ├── packages.js         # 5 个主推套餐
    ├── items.js            # 11 大类树形数据
    └── overview.js         # 总览页表格数据
```

### 5.3 关键实现要点

1. **数据模块**：JS 模块中使用工厂函数（`cat()`, `sub()`, `leaf()`, `perUnitLeaf()`）构建三层树形数据，无冗余中间节点
2. **递归组件**：`tree-node` 通过扁平化列表（`flatList`）渲染，父页面管理展开/折叠状态（`expandedIds`）
3. **全包价计算**：`cart.getCartTotal()` 收集所有有价格的父节点（category / subcategory / process_group），按深度降序检测全选状态。无冗余 process_group 的子类下叶子直接挂载，第一章/第二章保留进程组用于细分计费；全选子类时使用子类全包价，全选大类时使用大类全包价
4. **Canvas 渲染**：计算所需高度后绘制，导出为临时文件路径后用 `<image>` 展示，支持长按保存
5. **全局状态**：购物车使用 `app.globalData.cartItems` 存储，页面 `onShow` 时同步，组件通过 `setData` 传递

## 6. 非功能需求

### 6.1 性能
- 首屏渲染时间 < 1s
- 树节点懒渲染，默认展开前两级

### 6.2 兼容性
- 支持微信小程序基础库 ≥ 2.10.0
- 适配 iOS 及 Android 微信客户端

### 6.3 可用性
- 无需登录/注册即可使用
- 无需网络请求（纯本地数据 + 本地图片生成）
- 购物车数据页面切换时保持

### 6.4 安全性
- 无用户数据收集
- 无外部支付对接
- 无服务端交互

## 8. 附录

### 8.1 名词说明

| 术语 | 说明 |
| --- | --- |
| 叶子节点 | 树结构中最末端的节点，可直接购买的最小单位 |
| 全包优惠价 | 父节点下所有子项的打包价格，低于逐项累加 |
| 子类 | 大类下的分类层级（如"第一章"、"四号谷地"）；若无有意义细分，叶子直接挂大类下 |
| 套餐 | 跨品类的打包组合，含多项不同类别的服务 |

### 8.2 现有数据概览

- 套餐：5 个主推套餐（其中 2 个含变体：默认/不含全收集）
- 单项：11 大品类
